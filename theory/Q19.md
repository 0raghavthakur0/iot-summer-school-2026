# Q19 – analogWrite(), analogRead() and PWM

## 1. analogRead()

### Definition
`analogRead()` is used to read analog voltage from an analog input pin (A0–A5 on Arduino UNO).

### Range
0 to 1023 (10-bit ADC)

### Example

```cpp
int sensorValue = analogRead(A0);
```

### IoT Example
Reading the value from a soil moisture sensor to determine whether a plant needs watering.

---

## 2. analogWrite()

### Definition
`analogWrite()` is used to generate a PWM (Pulse Width Modulation) signal on PWM-supported digital pins.

### Range
0 to 255

### Example

```cpp
analogWrite(9, 128);
```

This produces approximately 50% duty cycle.

### IoT Example
Controlling the brightness of an LED or the speed of a DC motor.

---

## 3. What is PWM?

PWM stands for **Pulse Width Modulation**.

It is a technique used to simulate analog output by rapidly switching a digital signal ON and OFF. By changing the duty cycle, the average voltage supplied to a device can be controlled.

---

## 4. Why is PWM used?

- To control LED brightness.
- To control DC motor speed.
- To reduce power consumption.
- To simulate analog output using digital pins.

---

# Comparison Table

| Feature | analogRead() | analogWrite() |
|----------|--------------|---------------|
| Purpose | Reads analog input | Generates PWM output |
| Pin Type | Analog Pins (A0–A5) | PWM Pins (3, 5, 6, 9, 10, 11) |
| Range | 0–1023 | 0–255 |
| Return Value | Integer | No return value |

---

## Conclusion

`analogRead()` measures analog signals from sensors, while `analogWrite()` controls output devices using PWM. Both functions are essential for building IoT and embedded systems.
